# Mihomo Config Structure

## A. Classic Config (static proxies)

Generated by subscription converters or manually built. Contains inline proxy definitions.

```yaml
port: 7890              # HTTP proxy
socks-port: 7891        # SOCKS5 proxy
redir-port: 7892        # Redirect (TPROXY)
mixed-port: 7893        # HTTP + SOCKS5 mixed
allow-lan: false
mode: rule
log-level: info
external-controller: 0.0.0.0:9090   # Management API

proxies:
  - name: "🇭🇰 HK-01"
    type: trojan
    server: hk01.example.com
    port: 443
    password: "uuid-or-password"
    udp: true
    sni: www.example.com
    # ... other node definitions

proxy-groups:
  - name: "Proxy"
    type: select
    proxies:
      - "🇭🇰 HK-01"
      - "🇺🇸 US-01"
      - DIRECT

rules:
  - GEOIP,CN,DIRECT
  - MATCH,Proxy
```

## B. Proxy-Provider Config (dynamic subscription pull)

Preferred for large subscriptions that auto-update. Nodes are fetched from the subscription URL at startup and periodically.

```yaml
port: 7890
socks-port: 7891
mixed-port: 7892
mode: rule
log-level: info
external-controller: 0.0.0.0:9090
external-ui: ui

proxy-providers:
  my_provider:
    type: http
    url: "https://your-subscription-url/sub"
    interval: 3600             # re-fetch every hour
    health-check:
      enable: true
      url: http://www.gstatic.com/generate_204
      interval: 300

proxy-groups:
  - name: "Proxy"
    type: select
    use:
      - my_provider
    proxies:
      - "Auto"
      - DIRECT

  - name: "Auto"
    type: url-test
    use:
      - my_provider
    url: http://www.gstatic.com/generate_204
    interval: 300
    tolerance: 50

rules:
  - DOMAIN-SUFFIX,provider-domain.com,DIRECT   # bypass proxy for subscription fetch
  - GEOIP,CN,DIRECT
  - MATCH,Proxy
```

### Key differences from static config

| Aspect | Static (proxies:) | Proxy-Provider (proxy-providers:) |
|--------|-------------------|-----------------------------------|
| Node definition | Inline in config | Pulled from URL at runtime |
| Auto-update | Manual re-download | Automatic via interval |
| Subscription domain bypass | Not needed | REQUIRED — or deadlock |
| Config size | Large (all nodes inline) | Tiny (just provider URL + groups) |
| API node listing | Under `proxies/` in API | Under provider name in API |

### Verification that providers loaded

```bash
# Check API for proxy lists
curl -s http://127.0.0.1:9090/proxies | python3 -c "
import json,sys
data = json.load(sys.stdin).get('proxies', {})
nodes = [(k,v.get('type','')) for k,v in data.items() if v.get('type') in ('Trojan','Vmess','Vless','Shadowsocks','Hysteria2')]
print(f'Loaded {len(nodes)} proxy nodes')
for name, typ in nodes[:5]:
    print(f'  {name}: {typ}')
if len(nodes) > 5:
    print(f'  ... and {len(nodes)-5} more')
"
```

## C. Common Subscription Formats (base64-decoded)

Providers return base64-encoded newline-separated proxy URLs. Common protocol prefixes:

| Prefix | Format | Key params |
|--------|--------|------------|
| `trojan://` | `trojan://password@host:port?...&sni=X&peer=Y#Name` | password, sni, peer, allowInsecure |
| `vless://` | `vless://uuid@host:port?type=tcp&security=reality&pbk=X&sid=Y&fp=chrome#Name` | uuid, pbk, shortId, flow, sni |
| `vmess://` | Base64-encoded JSON: `{"add":"host","port":443,"id":"uuid","aid":0,"net":"ws","tls":"tls","host":"sni"}` | Whole structure base64-wrapped |
| `ss://` | `ss://method:password@host:port#Name` | method (chacha20-ietf-poly1305, aes-256-gcm) |

## D. Management via external controller

```bash
# Get version
curl http://127.0.0.1:9090/version

# List all proxies and groups
curl http://127.0.0.1:9090/proxies

# Switch node in a selector group
curl -X PUT http://127.0.0.1:9090/proxies/Proxy \
  -H "Content-Type: application/json" \
  -d '{"name":"🇭🇰 HKG·香港01"}'

# Get traffic stats
curl http://127.0.0.1:9090/traffic
```
